# Next.js 学習用サンプルコード集

本リポジトリは、Next.js と React の基礎から応用までを段階的に学習するためのサンプルコードを提供します。各ステップは独立した Next.js プロジェクトとして構成されており、順番に学習を進めることで、モダンな Web 開発に必要なスキルを習得できます。

## 📚 学習ステップ一覧

### Step 01: React の基礎とコンポーネント化

**ディレクトリ**: `step01_react1`
**学習内容**:

- React コンポーネントの基本構造
- コンポーネント指向のアプリケーション設計
- Next.js の初期設定と基本構成

### Step 02: Props によるデータ受け渡し

**ディレクトリ**: `step02_react2`
**学習内容**:

- Props を使用した親子コンポーネント間のデータ受け渡し
- コンポーネントの再利用性向上
- 単方向データフローの理解

### Step 03: React Hooks の活用

**ディレクトリ**: `step03_hooks`
**学習内容**:

- `useState`: コンポーネントの状態管理
- `useEffect`: 副作用の処理とライフサイクル
- `useMemo`: パフォーマンス最適化（メモ化）
- `useCallback`: 関数のメモ化

各フックの専用ページで実践的な使い方を学べます。

### Step 04: Next.js アプリケーション開発

**ディレクトリ**: `step04_create_nextjs`
**学習内容**:

- Next.js プロジェクトの作成方法
- アプリケーションの基本構造
- 開発環境のセットアップ

### Step 05: Next.js のコンポーネント設計

**ディレクトリ**: `step05_nextjs_app`
**学習内容**:

- App Router を使用したページ構成
- レイアウトコンポーネントの設計
- ファイルベースルーティング
- コンポーネントの階層構造

### Step 06: 様々なスタイリング手法

**ディレクトリ**: `step06_styling`
**学習内容**:

- Tailwind CSS を使用したスタイリング
- グローバルスタイルとコンポーネントスタイル
- レスポンシブデザインの実装
- Next.js における CSS の最適化

### Step 07: 外部 API 連携

**ディレクトリ**: `step07_fetch_data`
**学習内容**:

- fetch API を使用したデータ取得
- 非同期処理の扱い方
- REST API との連携
- データの表示とエラーハンドリング

### Step 08: React Server Components (RSC)

**ディレクトリ**: `step08_rsc`
**学習内容**:

- Server Components と Client Components の違い
- 静的レンダリングと動的レンダリング
- データフェッチングのベストプラクティス
- パフォーマンス最適化

以下のページを含みます:

- `/static`: 静的レンダリングの例

- `/products`: 商品一覧ページ

### Step 10: 入力フォームの基礎

**ディレクトリ**: `step10_input_form`

## 🚀 各ステップの使用方法

### 1. ディレクトリに移動

```bash
cd step01_react1  # 任意のステップディレクトリ
```

### 2. 依存関係のインストール（初回のみ）

```bash
npm install
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開いて確認できます。

### 4. その他の利用可能なコマンド

```bash
npm run build  # プロダクションビルド
npm run start  # プロダクションサーバーの起動
npm run lint   # ESLintによるコード検証
```

## 💡 学習の進め方

1. **順番に学習**: Step 01 から順番に進めることを推奨します。各ステップは前のステップの知識を前提としています。
2. **コードの理解**: 各ディレクトリ内のコードを読み、動作を確認しながら理解を深めましょう。
3. **実践的な修正**: コードを実際に変更して、動作がどのように変わるか試してみましょう。
4. **公式ドキュメント参照**: 不明な点は以下の公式ドキュメントを参照してください。
